# Правила форматирования

> Загружается всегда. Источник: CLAUDE.md § 6 (вынесено для сжатия контекста).

## Язык — русский (БЛОКИРУЮЩЕЕ)

Все ответы, метки, категории, заголовки — на русском. EN только в коде, терминах (DDD, BC, UL), именах файлов.
НЕ: HIGH / MEDIUM / LOW / CRITICAL / DONE. ДА: **критический**, **высокий**, **средний**, **низкий**, **выполнено**.

## Таблицы с РП

- Активные (pending, in_progress): название **жирным**
- Done: зачёркнуть строку без bold (`| ~~#~~ | ~~название~~ | ✅ | ~~done~~ |`). Эмодзи-статусы (✅, 🔄, ↗️, 📦) — **без** `~~`. Тильда `~` внутри `~~` ломает GitHub — заменять `~30 мин` → `30 мин`, `~/IWE` → `IWE`.

## Collapsible sections (>80 строк) — ТОЛЬКО В ФАЙЛАХ

**Файл-артефакт >80 строк** (исчерпывающий список: WeekPlan, DayPlan, Strategy, WP-context, спецификации) → `<details><summary><b>Название</b></summary>…</details>`; главная секция `<details open>`, без `---` между секциями.

**БЛОКИРУЮЩЕЕ — никогда в чате.** Ответ пилоту, синтез итогов, пост-отчёт — НИКОГДА не оборачивать в `<details>` (терминал не сворачивает HTML). Длинный ответ — обычный markdown или файл отчёта; `<details>` в чат — только по явному запросу пилота (усиливает A7/A7.1).
